Upgrades overview

upgradecomments
GRID system replaces former 'networks': a GRID is a PROC or a VIEW made up of NODES
NODES, replacing former 'STATIONS', are independent entities associated to one or more GRID(S)
GRIDS belongs to OWNERS (former 'OBSERVATOIRES') and DOMAINS (former 'DISCIPLINES')
-PROCS equivalent to former 'networks' whose 'net' > 0
-VIEWS equivalent to former 'networks' with 'ext' defined
-NODES replace former 'STATIONS'
-FORMS user's input system associated to PROCs (eg. legacy EAUX, FISSURO,...)
-DOMAINS former DISCIPLINES, now defined in a RDBMS (currently SQLITE)
Source Code / Development - SVN: WebObs source code (Perl, Matlab, Bash, Javascript, css) now SVN'd
- Cleanup (warnings, dead code/vars, standards, naming conventions, relocation)
- Python building blocks (webobs configuration interfaces)
- WebObs Perl Package: common subroutines repackaged and/or redesigned for new functionalities
Site layout / Directories Reorganized:
- 1st level: CODE/, CONF/, DATA/, OUTP/, OUTR/ Benefit from the configuration files (of the key|value form) 'substitution' feature:
- avoid hard-coded path+filename concatenations in programs whenever possible,
- better use one target path or filename variable, full path being defined in configuration file
Specific Configuration files former WEBOBS.conf exploded to several functionaly-dedicated configuration files
Implicitly import common structures in cgi Hash of Hashes of system-wide configuration files are automatically loaded/shared upon WebObs modules reference (use):
avoid reading these configuration files and standardize access/code.
These system-wide (common) structures are: %WEBOBS, %USERS, %DISCP, %OWNERS, %NODES, %GRIDS
Consolidated Read Conf. files functions Perl CGIs: readConf, readCfgFile, readFile:
- standardized configuration formats: [key|value] or [key|value...|values] formats
- variable's substitution (ie. full paths defined/built in *conf / *rc files rather than in code)
Matlab: readcfg:
- Perl CGIs formats compatibility: [key|value] or [key|value...|values] formats
- no matlab-like syntax for values; data type conversions as required by *.m's
File Locking - Perl CGIs Built-in 'shared flock' on read-file
- sysopen+flock method on write-file (see operationnal examples for file updates in most 'post*.pl')
User Authorization No more 'user level': files and/or functions requiring protection are represented by a 'resource' to which each (or group of) user(s) are given explicit authorization
Resources are defined as type/name. Types are:
authprocs, authviews, authforms, authwikis, authmisc
Users are granted R(ead), E(dit) or A(dmin) access right to a given resource
Authorizations are defined in a RDBMS (currently SQLITE). High level access to authorization checking is available to cgi developers: clientHas... like APIs
Jobs scheduler Perl daemon to schedule/monitor execution of user's scripts (periodic and/or on-demand) aka "jobs"
Notifications system Perl daemon to centralized events messages and trigger notifications (email and/or user's processes such as Nagios)
ISO8859 vs UTF8. Conversions
HTML / Javascript Changes, additions and re-write:
- Iframe replaces frameset, 'window manager' utilities (wm.js, wm.css)
- jQuery MarkItUp plugin for text/html/wifi
- Ajax post for forms submits, info from forms processing returned as javascript alerts
- formats, css
Migrating from existing webobs M2G.pm to help in migrating legacy code/structures
- BASE: 'reseaux' to 'VIEWS, PROCS and FORMS' + 'stations' to 'NODES' - Additional functions for syntax/formats/... - 'dry run' mode provided
Package and Installation Exporting WebObs code as a SVN working copy OR package.tgz
Installation procedure included in package:
- checking for dependencies - installing a 'default' (ie. model) WebObs configuration from package
